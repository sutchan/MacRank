# 03. 开发标准 (Development Standards) v0.6.1

## 1. 目录结构与扁平化规范
### 1.1 核心目录
所有逻辑组件应放置于：
- `app/components/`: UI 展现层。
- `app/hooks/`: React 自定义 Hooks，用于状态管理。
- `app/lib/`: 计算逻辑、跑分加权、类型定义。
- `app/services/`: 外部 API (Gemini) 封装。
- `app/locales/`: 10 国语言翻译字典。

**绝对禁令**: 严禁在根目录下创建 `src/` 文件夹。

## 2. 编码与样式规范 (Coding Styles)
### 2.1 TypeScript
- 强制开启 `strict` 模式。
- `MacModel` 接口为全局核心模型，任何数据变动必须通过该接口。
- 严禁使用 `any`。

### 2.2 样式守则
- 95% 的样式应通过 **Tailwind CSS Utility Classes** 实现。
- 动态注入的样式（如段位颜色）必须使用静态映射，禁止直接字符串拼接类名，以防构建丢失。

## 3. 测试与 QA 验收 (Testing)
### 3.1 核心验收点
- **数据精准度**: M4/M5 跑分误差不得超过行业预估值的 ±5%。
- **交互鲁棒性**: 悬浮对比条在移动端不得遮挡 AI 触发按钮。
- **多语言一致性**: 切换语言后，货币符号（¥/$/€）与数值必须正确转换。
