# 01. 核心规格 (Core Specification) v0.6.1

## 1. 项目定义与核心价值
### 1.1 核心使命
MacRank 致力于通过客观的 Geekbench 6 跑分数据，消除苹果营销术语带来的认知壁垒，为用户提供最直观的性能天梯与智能选购建议。

### 1.2 三大支柱
- **数据透明**: 拒绝主观评价，一切以归一化 Benchmark 为准。
- **动态场景**: 针对不同职业场景提供定制化的性能权重模型。
- **极致视觉**: 像素级对齐 macOS 原生审美（毛玻璃、SF Pro 字体）。

## 2. 功能组件规格 (Component Specs)
### 2.1 性能天梯表 (MacTable)
- **响应式行为**: 
  - **移动端**: 冻结首列“机型名称” (Sticky column)，其余列支持横向滚动。
  - **交互**: 单元格点击触发 `DetailModal`；复选框触发 `CompareBar`。
- **视觉反馈**: 悬浮行高亮，选中行显示侧边蓝色指示条。

### 2.2 性能雷达图 (Radar Chart)
对比模式下必须展示以下五个维度：
1. **单核性能**: 响应速度基准（SC/5000）。
2. **多核性能**: 生产力/编译基准（MC/35000）。
3. **图形性能**: 渲染/游戏基准 (MT/250000)。
4. **性价比 (Value)**: 综合得分 / 售价（归一化系数 15）。
5. **内存扩展性**: 最大支持内存容量（相对于 192GB 锚点）。

### 2.3 悬浮对比条 (CompareBar)
- **显示条件**: 选中机型数量 > 0 且对比弹窗关闭时。
- **限制**: 最多支持 2 款机型对比。
- **功能**: 一键清除、移除单项、发起对比。

### 2.4 分享逻辑 (Share System)
- **判定优先级**: `navigator.share` (原生分享) > `navigator.clipboard` (剪贴板复制)。
- **分享内容**: 包含当前筛选条件的 URL Hash 链接。

## 3. UI 设计语言
- **背景**: `gray-50` (Light) / `black` (Dark)。
- **玻璃拟态**: 容器使用 `backdrop-blur-md` 结合 80% 透明度。
- **段位视觉**: 
  - **S+**: 幻彩渐变 (`from-indigo-500 via-purple-500 to-pink-500`) 配合 Pulse 动画。
  - **S/A/B/C/D**: 遵循 Apple 系统色阶（紫/蓝/绿/黄/灰）。
